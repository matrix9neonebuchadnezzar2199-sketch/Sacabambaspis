☑ ID Time Log Desc Message 判定
🎓 解析官メモ: プロセス解析（5つの判別ルール） このタブでは、現在実行中の全プロセスを5つの観点で自動判別しています。

① 正規パス偽装 (Masquerading): svchost.exe等がSystem32以外にある場合を検知
② ファイル名偽装 (Typosquatting): 正規名と1-2文字異なるプロセスを検知
③ 親子関係の異常: 想定外の親プロセスからの起動を検知
④ 不審フォルダからの実行: Temp, Users\Public等からの起動を検知
⑤ コマンドライン解析: Base64エンコードや外部ダウンロード系を検知
☑ PID Name Parent Path Reason DNA Start Time 判定
🎓 解析官メモ: メモリ領域解析 🔴 危険:RWX（匿名メモリ）: ファイルレス攻撃で使用。最も警戒すべきサイン。
🟡 JIT: Chrome/Edge/Node.js等のJSエンジンが正常動作で使用するRWX領域。
🟢 正常: RWX領域が検出されなかったプロセス。
権限の見方: RWX=読書実行, RW=読書, RX=読実行。RWXが0なら健全。
☑ PID Process Address Regions Permissions Type 解説
🎓 解析官メモ: 永続化 (タスクスケジューラ / レジストリRun / WMI) タスクスケジューラ: SYSTEM権限で定期実行。攻撃者の永続化手法として最も多用されます。
レジストリRun: ログオン時に自動実行。HKLM/HKCUの4つのRunキーを検査します。
WMI: ファイルレスの高度な永続化。APT攻撃で使用されます。
判別ロジック: ① 攻撃ツール検知 ② LOLBin+不審引数(-enc, -hidden等) ③ LOLBin+不審パス(Temp等) ④ 不審パス単体 ⑤ 正常
🔴 LOLBin+Base64引数の組み合わせはファイルレス攻撃の典型パターンです。
☑ Type Name Command Reason 判定
🎓 解析官メモ: ネットワーク深層解析（8つの検知ルール） このタブでは全ネットワーク接続を8つの観点で自動判別しています。

① 攻撃ツール通信: 既知の攻撃ツール(mimikatz等)のネットワーク使用を検知
② LOLBins外部通信: PowerShell/CMD等の管理ツールが外部IPと通信
③ 危険ポート: C2/バックドアで頻用されるポートへの接続
④ 非標準ポート: 一般的でないポートでの外部ESTABLISHED接続
⑤ ビーコニング: 同一リモートIPへの複数接続（C2コールバックの特徴）
⑥ 大量接続: 1プロセスからの異常な接続数
⑦ アカウント異常: システムプロセスが通常と異なるユーザーで実行
⑧ 未署名プロセス: デジタル署名のないプロセスが外部と通信
☑ Proto Local Remote 逆引きDNS State Process User 署名 接続数 Reason 判定
🎓 解析官メモ: 実行痕跡 (Evidence) UserAssist: エクスプローラ経由で実行されたプログラムの履歴（ROT13エンコード）。ファイルが削除されていても実行記録が残ります。
Prefetch: プログラム実行時にOSが自動作成するキャッシュ。攻撃ツールのPrefetchが存在すれば過去に実行された証拠です。
5つの判別ルール: ① 攻撃ツール検知 ② LOLBin実行 ③ 偵察コマンド ④ 不審パスからの実行 ⑤ 正常判定
⚠ Prefetchの取得には管理者権限が必要です。
☑ Source Artifact Time
